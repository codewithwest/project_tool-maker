# v0.2.0 — Autonomous Pipeline & Database Persistence

**Date:** 2026-06-29

This release introduces a full PostgreSQL-backed pipeline with autonomous tool generation, a redesigned web UI, and comprehensive dependency management.

## Highlights

### Autonomous DB Pipeline
- End-to-end 6-stage pipeline: **Analyse → Plan → Validate → Implement → Test → Review**
- Generates test functions alongside implementations
- Auto-fix loop: up to 3 LLM-driven attempts when tests fail
- Fallback test resolution and direct-execution fallback strategies
- Stage-by-stage logging with rich formatting for UI streaming
- Persists tools, plans, executions, and reviews to PostgreSQL

### Database Layer
- PostgreSQL connection pool with auto-creation of missing databases
- Migration system with SQL files, rollback support, and status tracking
- 6 tables: `config`, `plans`, `plan_steps`, `tools`, `executions`, `reviews`
- Full CRUD operations: `save_tool()`, `get_tool()`, `list_tools()`, `delete_tool()`
- Foreign key relationships with `ON DELETE CASCADE`
- Auto-runs pending migrations on first connection

### Planner Module
- `Planner` — breaks user goals into step-by-step plans using LLM
- `PlanValidator` — validates plan structure with cycle detection and LLM review
- `PlanExecutor` — executes steps in topological order with dependency passing
- `ResultReviewer` — reviews execution results against original goals
- `WriterToFile` — writes results to disk with auto-naming

### Dependency Management
- `scan_imports()` — AST-based import extraction from tool code
- 200+ standard library module detection
- 50+ module-to-package name mappings (`bs4`→`beautifulsoup4`, `PIL`→`Pillow`, etc.)
- Auto-install via `uv pip install` with 120s timeout
- Approval workflow: configurable approved list, auto-approve mode, or per-prompt
- Sandbox auto-retry: retries up to 3 times with auto-install on `missing-dep:` errors

### Redesigned Web UI
All pages rebuilt with a modern glass-morphism aesthetic (inspired by Linear × Vercel × GitHub Actions):

- **Dashboard** — gradient hero, card grid, live terminal panel with macOS-style dots
- **Pipeline** — gradient hero, segmented progress bar, pill-shaped stage chips, terminal panel with LIVE badge, celebratory outcome card
- **Execute** — IDE-style editor shell with collapsible left sidebar (Saved + Database tools), command bar with gradient execute button, tabbed result panel (Output / Explain / Refine), pulse loaders
- **Analyze** — gradient hero, metric cards (modules/dependencies/entry points), glass table display
- **Provider** — dual-panel config + models display, test prompt form
- **Config** — gradient hero, glass card form sections, migration status viewer
- **Docs** — tabbed navigation across release notes and README, client-side markdown rendering via marked.js, AJAX content loading with no page reloads

20+ REST API endpoints including:
- `/api/fix` — iterative tool fixing via LLM
- `/api/refine` — refine tools with user instructions
- `/api/explain` — LLM-generated explanations of tool execution
- `/api/deps/check` and `/api/deps/approve` — dependency management
- `/api/db/migrations` and `/api/db/migrate` — migration management
- `/api/delete-db` — delete database tools

### Tool Fixer
- `ToolFixer` class for iterative LLM-powered code repair
- Reads, tests, and fixes tools with configurable max attempts (default 3)
- Structured fix prompts requesting minimal changes with type annotations
- Works with both file paths and code strings

### Enhanced CLI
- `config show|set|whitelist|approve-dep|auto-approve` — full config management
- `dep install|sync|approve|scan` — dependency management
- `fix <file>` — fix broken tools via LLM
- `migrate up|rollback|status` — database migration management
- `pipeline <goal>` — run full DB-backed pipeline from the terminal

### Sandbox Improvements
- Missing-dependency auto-retry with dynamic install
- Module purging on startup to prevent import smuggling
- `##SBX##` marker protocol for reliable JSON output extraction
- Configurable extra whitelist passed through the execution chain
- Enhanced safe built-in set (added `print`, `getattr`, `hasattr`, `setattr`, `callable`, `type`)

### Async API
- Async `execute_tool_async()` and `generate_tool_async()` on ToolMaker
- Async `generate()` on LLMProvider for non-blocking LLM calls

### CI/CD
- GitHub Actions CI with matrix testing (Python 3.10–3.14)
- Ruff linting, pytest, and package build checks
- Release workflow with PyPI publishing (trusted publishing) and auto-changelog

### Other Improvements
- `psycopg2-binary` and `flask` promoted to core dependencies
- `python-dotenv` removed in favor of built-in `dotenv.py`
- Default database DSN updated with `sslmode=require`
- Fallback name generator now filters articles (`a`, `an`, `the`)
