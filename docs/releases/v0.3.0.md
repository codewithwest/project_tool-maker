# v0.3.0 ‚Äî Zero-Setup, Pluggable Providers, Remote Client & Docker

**Date:** 2026-06-29

This release focuses on lowering friction to zero: SQLite fallback so PostgreSQL is optional, Docker Compose for one-command deployment, pluggable LLM providers beyond Ollama, a remote API client for headless integration, and a streamlined onboarding flow.

## Highlights

### Dual Database Backend
- **SQLite (stdlib)** is now the default ‚Äî zero dependencies, zero config
- PostgreSQL remains fully supported (opt-in via `TOOLMAKER_DB_DSN` env var)
- Auto-detection: empty/unset DSN ‚Üí SQLite, `postgresql://` DSN ‚Üí PostgreSQL
- Separate migration SQL files per backend (`*.sqlite.sql` / `*.postgres.sql`)
- `psycopg2-binary` moved to optional `[postgres]` extra ‚Äî core install is leaner

### Pluggable LLM Providers
- **OpenAI** ‚Äî uses the `openai` package (optional), supports GPT models and any OpenAI-compatible endpoint
- **Anthropic** ‚Äî uses the `anthropic` package (optional), supports Claude models
- **Ollama** ‚Äî unchanged, still the default with zero external deps
- Provider selected via `ToolMaker(llm_provider="openai", ...)` or env var

### ToolMakerClient (Remote API Client)
- `ToolMakerClient(base_url)` ‚Äî consume Tool Maker as a REST service
- No local Ollama or DB required ‚Äî talks to a running server
- Methods: `list_tools()`, `execute()`, `explain()`, `refine()`, `fix()`, `run_pipeline()`, `analyze()` and more

### Docker Compose
- `docker-compose.yml` with three services: Ollama, PostgreSQL, and the app
- `Dockerfile` for the app image
- One-command startup: `docker compose up`

### Onboarding (`init` Command)
- `llm-tool-maker init` ‚Äî interactive setup that checks Ollama, creates config directory, runs DB migrations, and writes a `.env` file
- Supports `--db postgres --url <dsn>` for PostgreSQL setup

### API-Only Mode
- `llm-tool-maker ui --api-only` serves only REST API endpoints (no Jinja templates)
- Useful for headless/backend integrations

### Enhanced Public API
- `tool_maker.client.ToolMakerClient` ‚Äî remote HTTP client
- `tool_maker.db.backends.SqliteBackend` / `PostgresBackend` ‚Äî importable backends
- `tool_maker.llm.provider.OpenAIProvider` / `AnthropicProvider` ‚Äî direct provider access
- `tool_maker.set_backend()` ‚Äî override backend in tests/programs
- Updated `__all__` exports with all new symbols

### Other Improvements
- `psycopg2-binary` moved from core to `[postgres]` optional extra
- Default `TOOLMAKER_DB_DSN` changed from `postgresql://...` to empty string (SQLite)
- `TOOLMAKER_DB_PATH` env var for configuring SQLite file location
- Ruff linting passes clean on all new code
